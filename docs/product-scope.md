# 产品范围

## 产品定位

Codex Desktop Linux 是 Codex app-server 协议的原生 Linux 桌面客户端，面向长期使用 Codex 完成代码分析、修改、审查和自动化任务的开发者

产品在视觉结构、信息密度和核心交互上参考官方 Codex Desktop，同时补充 Linux 跨发行版、多服务器、远程工作区和文件预览能力

客户端不是独立智能体实现，不在前端复制 Codex 的推理、工具执行或审批逻辑，会话、回合、工具活动、审批请求和账户限额均以 app-server 为事实来源

## 目标用户

| 用户 | 主要诉求 |
| --- | --- |
| 本机开发者 | 在本地项目中启动 app-server 并完成日常编码任务 |
| 远程开发者 | 连接开发机或容器中的 WebSocket app-server |
| 多项目用户 | 同时在多个窗口处理不同服务器和工作区的会话 |
| 审查者 | 查看历史修改、命令输出、计划和审查结果 |
| 高强度用户 | 使用键盘快速切换会话、模型、命令、文件和技能 |

## 核心任务

1. 选择或新建服务器并连接
2. 选择工作目录或恢复已有会话
3. 输入任务并配置模型、思考程度、审批与沙箱策略
4. 查看可恢复的回答和工具活动，并在独立区域关注瞬时状态
5. 处理审批、查看文件、复制内容、分叉会话或继续追问
6. 切换窗口、服务器或会话后恢复一致状态

## 产品目标

- 在主流 Linux 发行版上提供接近官方 Codex Desktop 的编码智能体体验
- 支持本机 stdio app-server 和远程 WebSocket app-server
- 支持直连、HTTP CONNECT、SOCKS5 和 SSH 隧道远程连接路径
- 支持保存多个服务器，并在多个独立窗口中连接不同服务器或工作区
- 准确恢复可持久化的会话、回合、消息和工具活动
- 清晰展示连接状态、审批请求、任务运行状态和账户剩余限额
- 在客户端内安全预览 Markdown、代码、图片、文本文件和远程文件引用
- 保持前后端职责清晰，使协议、界面和 Linux 交付能够独立演进

## 非目标

- 不实现 Codex 模型服务、推理引擎或工具执行沙箱
- 不修改 Codex app-server 协议或上游 Codex 仓库
- 不以浏览器部署或移动端适配作为首个版本目标
- 不在首个版本中实现团队管理、组织计费和集中策略下发
- 不将系统托盘作为首个版本目标
- 不复刻官方客户端的内部私有接口、实验入口或 macOS 专属系统集成
- 不为旧版协议添加未经证实的字段猜测、别名映射或兜底分支

## 发布优先级

### P0 首个可用版本

- Tauri Linux 桌面壳与单实例窗口管理
- 本机 stdio 和远程 WebSocket 服务器配置
- 直连、HTTP CONNECT、SOCKS5 和 SSH 隧道连接路径
- 多服务器保存、连接、编辑、删除和在新窗口打开
- app-server 初始化、通知、请求、断线和重连生命周期
- 会话新建、恢复、归档、删除、分组和分页加载
- 消息流、工具活动、审批、计划、差异和命令输出
- `/`、`@`、`$` 输入建议菜单
- Markdown 安全渲染和常见远程文件预览
- 账户剩余限额圆环和详情浮层
- 模型、思考程度、权限和工作目录配置
- AppImage、deb 和 rpm 交付物

### P1 增强版本

- PDF 内嵌预览
- Flatpak 包
- 应用与插件浏览入口
- 会话全文搜索和高级筛选
- 桌面通知细分设置
- 多窗口布局记忆和跨窗口会话拖放
- 更完整的无障碍自动化测试

### P2 探索版本

- 自定义主题和高对比度主题
- 多栏会话对照
- 可配置命令和快捷键
- 企业级集中服务器配置
