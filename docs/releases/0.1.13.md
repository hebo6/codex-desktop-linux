# Codex Desktop Linux 0.1.13

This release makes long-running commands visible and controllable after their originating turn has finished

## Highlights

- Shows a persistent running-command panel above the composer with command count, elapsed time, working directory, and latest output

- Locates the original command activity in conversation history without moving completed content

- Terminates an individual background terminal without interrupting the current turn

- Keeps activity groups visibly running while a command remains active

- Shows background-command counts in the conversation sidebar

- Restores authoritative background-terminal state after reconnecting or switching conversations

## Packages

Signed AppImage, deb, and rpm packages are provided for x86_64 and aarch64

Verify `SHA256SUMS.asc`, then verify the selected package's detached `.asc` signature and checksum before installation

The release signing key fingerprint is `054F 6D64 D67D 4ED8 FB36 384F 3A16 C998 1577 619D`

Download `codex-desktop-linux-release-key.asc`, `SHA256SUMS`, `SHA256SUMS.asc`, the selected package, and its matching `.asc` file, then verify them with GnuPG and sha256sum

```bash
gpg --show-keys --with-fingerprint codex-desktop-linux-release-key.asc
gpg --import codex-desktop-linux-release-key.asc
gpg --verify SHA256SUMS.asc SHA256SUMS
sha256sum --check SHA256SUMS
gpg --verify codex-desktop-linux-0.1.13-x86_64.AppImage.asc codex-desktop-linux-0.1.13-x86_64.AppImage
```

## System requirements

- x86_64 or aarch64 Linux with glibc 2.35 or newer

- X11 or Wayland desktop session

- deb and rpm packages require GTK 3 and WebKitGTK 4.1 from the distribution

- Linux Secret Service is recommended for credential persistence; when unavailable, the application requires confirmation before storing a credential in a plaintext file restricted to the current user

The AppImage is built on Ubuntu 22.04 and carries its WebKit and GStreamer runtime dependencies; systems without FUSE can use the AppImage extract-and-run mode

## Validation baseline

- Ubuntu 22.04

- Debian 12

- Fedora stable

- Arch Linux rolling

The automated release gate installs the x86_64 artifacts on all four distributions, launches the packaged WebView under controlled X11 and headless Wayland sessions, validates desktop integration, and verifies deb and rpm package removal

The aarch64 AppImage, deb, and rpm artifacts are built and tested on an Ubuntu 22.04 ARM runner

## Protocol baseline

The app-server protocol is generated from upstream commit `ac3da4fb1a2ad0ee2f0c867bfa81a5a3a3737f9c` with `experimentalApi` enabled

Remote WebSocket transport remains experimental upstream and should only be used with a trusted TLS-protected endpoint
