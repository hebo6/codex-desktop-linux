## 设计依据

界面设计以本文档、[产品范围](../product-scope.md)和[视觉设计参数](../visual-design.md)为依据

项目自身生成的确定性界面截图位于 `tests/visual/baselines`，用于验证会话主界面、斜杠菜单、模型菜单和设置页在不同尺寸与主题下的表现

视觉设计不复制其他平台的窗口装饰、系统字体或平台专属能力

## 设计原则

- 已完成的历史内容原则上保持不变，后续变化优先通过追加记录表达
- 进度条和圆环的高亮部分表示剩余量，配套数值也表示剩余量
- 常态信息使用低强调色，仅在需要关注或操作时使用显眼状态色
- 远程内容、外部链接和敏感配置必须在界面中呈现清晰的安全边界
- 鼠标操作必须具有键盘等价入口
- 状态变化不得造成内容裁切、挤压、重排跳动或短暂溢出
- 在可滚动容器内，各层级已展开项的标题栏滚动至对应层级的置顶位置时，应按层级依次粘性置顶，子级标题排列在父级标题下方，并保持可见，直到同层级下一项的标题栏滚动至该位置
- 默认跟随系统浅色或深色主题，并允许用户显式固定主题
- 侧边栏和主内容区使用轻微背景层级区分，不用高对比边框围住全部区域
- 主内容区左上角使用圆角，形成主内容覆盖侧栏背景的空间关系
- 选中态和键盘焦点态均不使用描边或外发光，选中反馈使用低强调背景、图标或位置指示表达
