# 键盘、无障碍与动画

## 快捷键对话框

按 `Ctrl+/` 打开键盘快捷键对话框

对话框居中显示标题、关闭按钮、搜索框和按功能分组的快捷键列表，内容超出可用高度后在对话框内部滚动

搜索同时匹配功能名称和快捷键，结果保留原分组顺序，没有匹配项的分组隐藏

同一功能存在多个快捷键时分别显示按键标签，不合并为单个文本标签

按 Esc 或点击关闭按钮关闭对话框，关闭后焦点回到打开前的元素

## 焦点

所有悬浮操作都提供键盘焦点等价入口

键盘焦点必须与悬浮态和选中态清晰区分，并通过低强调背景、图标或位置指示表达

模态对话框使用焦点陷阱，关闭后焦点回到触发元素

新消息到达不得强制移动焦点

审批出现时通过实时区域播报，但不抢占正在输入的光标

## 无障碍

正文和交互控件必须具有足够的对比度

首次使用 Tab 进入界面时提供跳到主内容入口

纯图标按钮必须有可访问名称和 tooltip

每个消息回合向辅助技术提供角色、内容和操作组语义

消息文字方向根据自身内容自动判断，不依赖应用当前语言

连接、限额和活动状态不得只依赖颜色

粗指针环境的交互命中区域至少为 44px，功能不得依赖悬浮

页面放大后核心功能仍可用，不出现页面级水平滚动

## 动画与反馈

容器展开时先完成尺寸动画，再显示新增内容

容器收起时先隐藏收起态无法容纳的内容，再执行尺寸动画

尺寸进度和内容可见性独立变化，避免裁切、挤压和跳动

系统启用减少动画后移除位移和缩放，只保留必要状态切换

系统启用减少透明度后取消背景模糊、遮罩渐变和半透明叠层，改用不透明表面与低强调分隔线

浮层以触发元素为变换原点，通过透明度和轻微缩放进入或退出，不得影响底层布局尺寸

新消息淡入不得改变最终占位尺寸

会话归档和删除先锁定行高，再淡出并收起，其他行平滑补位

限额圆环只动画弧长和颜色，不持续旋转

运行状态可以使用低强调脉冲，审批和错误使用静态高关注标识

所有可提交操作立即显示按下或忙碌反馈
