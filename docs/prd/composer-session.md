# 输入与会话配置

## 布局与文本编辑

输入区固定在主内容区底部并水平居中，消息流底部保留等高安全空间

新建页面的项目选择器独立位于问题输入框上方，与输入框水平对齐并保持紧密的上下空间关系；项目选择器所在容器仅上方使用圆角，下方保持直角并与问题输入框衔接

编辑器使用大圆角卡片容器，浅色主题使用白色背景、浅色边框和柔和阴影，深色主题使用略亮于页面背景的深灰色表面

无附件和结构化令牌时，编辑器分为上方多行文本区和下方操作栏；附件区和结构化令牌区出现时插入两者之间，各区域独立布局

操作栏左侧依次提供添加内容入口和审批模式选择，右侧依次显示当前模型、模型切换入口和圆形主操作按钮

主操作按钮使用向上箭头图标，空输入且无附件时降低强调度并禁用，存在可发送内容时切换为高对比样式

宽度充足且内容较少时编辑区与两侧操作紧凑排列，空间不足或编辑器展开时编辑区独占一行，配置和操作移动到底栏

宽窗口使用更宽的左右边距，窄窗口相应收紧

编辑器达到最大高度后内部滚动，滚动编辑器不得带动消息流

配置标签因空间不足折叠为图标时，必须保留完整可访问名称和 tooltip

输入区上方可以使用低强调渐隐层表达内容延续关系，渐隐层不得拦截选择、滚动或点击

编辑器支持纯文本、多行、撤销重做、输入法合成、结构化令牌和拖放附件

Enter 发送，Shift+Enter 换行，输入法合成期间 Enter 不触发发送

空输入且无附件时发送按钮禁用

草稿按窗口、服务器和临时会话自动保存，成功发送后清空

## 添加内容菜单

点击添加内容入口在输入框上方打开独立浮层，不改变编辑器尺寸，再次点击入口或点击浮层外部关闭

菜单提供添加图片和项目引用入口，常用提示词按本文件对应章节接入

添加图片打开本机图片选择器，项目引用在当前光标或选区位置插入 `@` 并立即打开引用建议菜单

选择项目引用后焦点回到编辑器，提交中或正在准备附件时禁用添加内容入口

## 结构化输入

| 输入类型 | 编辑器表现 |
| --- | --- |
| `text` | 普通文本 |
| `image` | 已上传图片附件 |
| `localImage` | 本机图片附件 |
| `skill` | 技能令牌 |
| `mention` | 文件、应用或插件引用令牌 |

结构化令牌显示为不可拆分的 inline chip，退格第一次选中整个令牌，第二次删除

## 运行中行为

当前回合运行且编辑器为空时，主操作按钮显示停止

用户输入新内容后，主操作按钮显示追加，旁边保留独立停止按钮

追加内容作为 steer 输入，最终消息结构以服务端返回为准

停止后立即显示正在停止并禁用重复点击，长时间未确认时显示仍在等待服务端

## 附件

支持文件选择、拖放、剪贴板粘贴和 `@` 菜单添加附件

图片展示缩略图、名称和移除按钮，普通文件展示类型图标、名称和大小

不支持的类型或大小在发送前显示具体错误，失败项保留在编辑器中

移除附件只修改草稿，不删除本机或服务器文件

## 常用提示词

添加内容菜单提供全局共享的常用提示词入口，入口说明应明确选择后立即发送

点击入口打开独立选择浮层，浮层支持按名称和内容搜索，每项展示名称、内容预览和发送图标，选择后将提示词作为独立用户消息立即发送

发送常用提示词不得读取、修改或清空编辑器中的文字、结构化令牌和附件，发送完成后恢复编辑器原有光标与选区；新会话首次发送常用提示词时，已有草稿迁移到新会话并继续保留

常用提示词沿用当前工作目录、模型、思考程度和权限配置；当前回合运行时作为追加内容发送，提交或停止期间不可发送

选择浮层底部提供管理入口，空列表直接提供新建入口；管理对话框支持搜索、新建、查看、编辑、删除和手动排序，删除前需要确认

提示词名称去除首尾空白后必须唯一且不超过 80 个字符，内容保留原始空白、去除首尾空白后不得为空且不超过 32000 个字符

常用提示词持久化在客户端全局存储中，所有服务器、项目和应用窗口共享；多个窗口并发修改时不得静默覆盖较新的内容

## 发送反馈

首次发送任务时才产生服务端会话，未发送的空白页不出现在会话列表

发送后编辑器立即进入提交中状态，消息在服务端确认后进入历史

请求结果不确定时显示正在核对状态，确认服务端结果后再决定是否清空草稿

## 输入建议菜单

### 通用行为

光标位于可触发位置并输入 `/`、`@` 或 `$` 时，在光标附近显示对应菜单

菜单作为独立浮层显示，不改变编辑器尺寸或挤压输入内容

菜单优先在输入框上方展开并与输入框左右边缘对齐，空间不足时向下展开，宽度和高度不得超过窗口可用区域，内容超出后内部滚动

菜单项展示图标、名称、说明、来源和可选快捷键

上下方向键移动选择，Enter 确认，Tab 确认并继续输入，Esc 关闭

鼠标移动只改变悬浮项，不抢占编辑器焦点，选择后焦点回到原光标位置

关闭菜单不删除已经输入的触发字符，输入法合成期间不触发菜单

### `/` 命令菜单

斜杠只在行首或前一字符为空白时触发，后续文本用于筛选名称和说明

| 命令 | 行为 |
| --- | --- |
| `/review` | 进入代码审查流程 |
| `/compact` | 请求压缩当前上下文 |
| `/continue` | 继续最近可继续的任务 |
| `/goal` | 创建或查看目标流程 |
| `/init` | 初始化项目指导文件 |
| `/mcp` | 查看 MCP 工具和资源 |
| `/plan` | 切换或请求计划模式 |
| `/settings` | 打开设置窗口 |
| `/feedback` | 打开反馈入口 |
| `/attach` | 打开附件选择器 |

不可用命令保留显示并解释原因

立即执行型命令选择后直接执行，参数型命令插入命令和空格后继续输入

### `$` 技能菜单

技能项显示名称、简短说明、来源和启用状态，并支持按名称和描述筛选

选择技能后插入显示为 `$技能名` 的结构化令牌

技能列表加载中显示骨架项，失败时不阻塞普通文本输入并提供重试

已禁用或当前服务器不可用的技能不可选择，并展示原因

### `@` 引用菜单

引用菜单聚合工作区文件、已知应用和插件引用，默认优先展示当前工作区最近文件

结果按文件、应用和插件分段，同名文件通过相对路径区分

选择文件后插入结构化引用令牌，可附加类型可以进一步选择只引用路径或附加内容

远程服务器不支持文件搜索时显示能力缺失说明，不扫描本机同名目录作为替代

## 模型、项目与权限

输入区底栏的默认值展示与请求覆盖状态独立建模，只有用户显式选择的模型、思考程度或权限进入发送请求，展示服务器默认具体值不得将其写入请求字段

### 模型与思考程度

输入区底栏显示当前模型短名称和思考程度，点击打开模型设置浮层

没有显式选择模型时，根据 `model/list` 中的 `isDefault` 展示 `默认 · {模型名称}`，服务端没有明确标记目录默认模型时仅显示服务器默认模型

没有显式选择思考程度时，展示当前目录模型的 `defaultReasoningEffort`，并保留服务端原始编码，例如 `medium`、`high` 或 `xhigh`

模型列表展示名称、说明、支持能力和推荐标记，当前项使用选中图标

切换模型只影响后续回合，不重写历史信息

思考程度显示服务端提供的选项，以及对速度、消耗和任务复杂度的简短影响

当前模型不支持的思考程度隐藏或禁用

### 项目

项目选择器仅在尚未创建会话的新建页面显示，首次发送并成功创建会话后隐藏，打开任何已有会话时也不显示

项目选择器显示文件夹图标和当前项目名称，悬浮后显示完整工作目录，点击打开项目下拉框

本机服务器使用系统目录选择器，远程服务器使用服务器侧路径选择或文本输入

选择项目决定新会话的工作目录

会话创建后项目保持固定，需要使用其他项目时新建会话

### 审批和沙箱策略

权限菜单展示当前审批策略、沙箱模式和网络能力

没有显式选择时，仅在 `config/read` 或 `configRequirements/read` 明确返回且管理要求允许该默认权限时显示 `默认 · {权限名称}`

服务端没有明确返回默认权限、配置读取失败或管理要求无法确认配置有效时显示默认权限，不根据工作目录信任状态或服务器内置规则在客户端推测具体值

选项说明以用户可理解的影响为主，高风险配置使用显眼警告

策略修改只影响后续回合，服务端拒绝时恢复原选择并展示原因
