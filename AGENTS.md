# 相关仓库

- Codex 源码仓库通过 `CODEX_SOURCE_DIR` 指定，需要了解 `app-server` 协议细节时可参考该仓库源码
- Codex 端及 `app-server` 协议仅供参考，禁止修改

# 开发原则

- 修改时不考虑向后兼容或添加兜底机制，应从长期可维护性和演进方向出发
- 技术选型应采用符合行业标准且前沿的方案
- 默认不更新视觉基线，除非用户明确要求

# 设计原则

- 界面设计与交互实现遵循 [产品需求文档](docs/prd/README.md)
- 主页面中已完成的历史内容原则上不应修改，这是设计层面的软性指导，不作为数据层硬约束，后续变化优先通过追加新记录表达
- 对表示已用与未用关系的进度条，高亮部分表示剩余量；若同时展示数值，数值也应表示剩余量
- 颜色强调程度应匹配信息的关注优先级：常态使用低强调色，仅在需要用户关注或采取行动时使用显眼的状态色

# 动画设计

- 容器由小变大时，先完成尺寸动画，再显示新增或仅在大形态下可见的内容
- 容器由大变小时，先隐藏小形态无法容纳的内容，再执行尺寸动画
- 容器尺寸进度与内部内容可见性应独立建模，内容进入由展开完成状态驱动，内容退出由收缩目标状态驱动
- 动画应保持空间关系连续，避免内容裁切、挤压、重排跳动或短暂溢出
